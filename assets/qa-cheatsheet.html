Severity & Priority Matrix
🔴 Blocker
App crash, data loss, cannot proceed. Stops all testing. Fix immediately.
🟠 Critical
Core feature broken, no workaround. Major business impact.
🟡 Major
Feature broken but workaround exists. Significant user impact.
🔵 Minor
Feature partially broken, minor impact. Low-risk defect.
⚪ Trivial
Cosmetic only. Typos, minor UI misalignment. No functional impact.
Priority vs Severity
High Sev / Low PriLogin broken on old browser no one uses
Low Sev / High PriCEO's name misspelled on homepage

Web Testing Checklist
  • UI consistent across pages (fonts, colors, spacing)
  • All links work — no 404s
  • Form validation (required fields, formats, limits)
  • Error messages are clear and helpful
  • Responsive on desktop / tablet / mobile
  • Cross-browser: Chrome, Firefox, Safari, Edge
  • Page load time < 3s (LCP, FCP, TTI)
  • Images have alt text
  • HTTPS only, no mixed content
  • Session timeout and re-login behavior
  • Copy-paste, back button, refresh behavior
  • Special characters in inputs (XSS, SQL injection)
  • Correct data displayed after CRUD actions
  • Pagination, sorting, filtering work correctly
📱
Mobile Testing Checklist
  • Multiple screen sizes (small, medium, large, tablet)
  • iOS and Android (latest + 1 version back)
  • Portrait and landscape orientation
  • Gestures: tap, swipe, pinch-to-zoom, long press
  • Keyboard does not obscure inputs
  • App permissions: camera, location, notifications
  • Interruptions: calls, SMS, notifications mid-flow
  • Background / foreground switch behavior
  • Low battery, low storage scenarios
  • Network conditions: 4G, 3G, offline, flaky WiFi
  • Auto-rotate, font size accessibility settings
  • Deep links and push notification navigation
  • App update backward compatibility

Accessibility (a11y)
WCAG Levels
AMinimum — must pass
AAStandard — most products target this
AAAEnhanced — aspirational
Core Checks
  • All images have descriptive alt text
  • Color contrast ratio ≥ 4.5:1 (text) / 3:1 (large)
  • Full keyboard navigation (Tab, Enter, Esc, Arrow keys)
  • Focus indicator visible at all times
  • Forms have labels linked via for/id
  • Error messages announced to screen readers
  • No content flashing >3x per second
  • Page has logical heading structure (h1→h2→h3)

Flaky Test Fixes
  • Use explicit waits, not sleep()
  • Isolate test data — no shared state
  • Run in parallel with independent environments
  • Retry on failure (max 2x) + log root cause

Classic Edge Cases to Always Try
  • Empty / null / blank inputs
  • Maximum length strings, numbers at limits
  • Special chars: <> " ' ; % & # @
  • Unicode, emoji, RTL text (Arabic, Hebrew)
  • Concurrent / duplicate submissions
  • Network drop mid-action
